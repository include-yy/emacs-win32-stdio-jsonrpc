* A simple JSON-RPC implementation in C++ for Emacs

A lightweight, header-only JSON-RPC 2.0 server implementation for Windows.

It serves as a bridge between Emacs and Windows-specific capabilities, such as hosting Microsoft Edge WebView2 content, by communicating over standard input/output (Stdio) using LSP-style framing.

It uses [[https://github.com/nlohmann/json][nlohmann/json]]([[https://github.com/nlohmann/json/releases/tag/v3.12.0][version 3.12.0]]) to parse JSON string.

*Key Features*

- *LSP-style framing*: Use =Content-Length: {LEN}\r\n\r\n{BODY}=.
  - ensure compatibility with Emacs's =jsonrpc.el=.
- =\r\n= Safety: Force stdio into =_O_BINARY= mode on Windows
  - prevent =\r\n= translation from breaking byte counts.

While primarily optimized to solve Windows-specific I/O quirks, the core JSON-RPC 2.0 logic — including parsing, state management, and message dispatching — is implemented in standard, platform-agnostic C++. It can be readily used on Linux or macOS (Not yet extensively tested on these platforms).

** Getting Started

*Prerequisites*

- ISO C++20
- nlohmann/json v3.12.0

Then, simply copy =json.hpp= and =jsonrpc.hpp= from =include= directory into your project's include directory.

*C++ Server Side*

The =Conn= object requires a Waker --- a callback triggered when a new message arrives in the queue.  In a Win32 application, use this to wake up your main message loop.

#+begin_src cpp
  #include "jsonrpc.hpp"
  #include <windows.h>

  #define WM_JSONRPC_WAKEUP (WM_USER + 1)

  int main() {
      uint32_t main_thread_id = GetCurrentThreadId();

      // 1. Define a Waker to signal the main thread from the reader thread
      auto waker = [main_thread_id]() {
          PostThreadMessage(main_thread_id, WM_JSONRPC_WAKEUP, 0, 0);
          };

      jsonrpc::Conn server(waker);

      // 2. Register a synchronous method
      server.register_method("add", [](const jsonrpc::json& params) {
          return params[0].get<double>() + params[1].get<double>();
          });

      // 3. Register a asynchronous method (non-blocking)
      // Useful for heavy tasks that should not freeze the main message loop.
      server.register_async_method("heavy_task", [](jsonrpc::Context ctx, const jsonrpc::json& params) {
          // Move task to a background thread; 'ctx' and 'params' are captured by value
          std::thread([ctx, params]() mutable {
              std::this_thread::sleep_for(std::chrono::seconds(3));
              ctx.reply("Task Complete!"); // Safely reply from any thread
              }).detach();
          });

      // 4. Register an exit notification (no response)
      server.register_notification("exit", [](const jsonrpc::json&) {
          PostQuitMessage(0); // Signal the GetMessage loop to terminate.
          });

      // Start the background I/O reader thread.
      server.start();

      // 5. Standard Win32 Message Loop
      MSG msg;
      while (GetMessage(&msg, nullptr, 0, 0)) {
          if (msg.message == WM_JSONRPC_WAKEUP) {
              // Process the message queue on the main thread
              server.process_queue();
              // Check if the reader thread died.
              if (!server.is_running()) {
                  break;
              }
          } else {
              TranslateMessage(&msg);
              DispatchMessage(&msg);
          }
      }
      // 6. Graceful Shutdown Implementation
      // Note: CancelIoEx provides a non-destructive way to interrupt a blocking stdin read.
      HANDLE hIn = GetStdHandle(STD_INPUT_HANDLE);
      if (hIn != INVALID_HANDLE_VALUE) {
          CancelIoEx(hIn, nullptr); // Forcefully abort pending I/O on the reader thread
      }
      server.stop();
      return 0;
  }
#+end_src

*Emacs Client Side*

When connecting from Emacs, *must* set the coding system to binary.  Failure to do so will cause Emacs to convert =\n= to =\r\n=, corrupting the Content-Length calculation.

#+begin_src elisp
  (require 'jsonrpc)

  ;; Change to example.exe's path if needed.
  (setq path (expand-file-name "../out/build/x64-Debug/example.exe"))

  (setq rpc (let ((proc (make-process :name "example"
                                      :command `(,path)
                                      ;; binary is necessary on Windows
                                      :coding 'binary) ))
              (make-instance 'jsonrpc-process-connection
                             :name "jsonrpc-example"
                             :process proc)))

  (jsonrpc-request rpc "add" [1 2])
  ;;=> 3.0
  (jsonrpc-request rpc "heavy_task" nil)
  ;;=> Wait 3 seconds
  (jsonrpc-notify rpc "exit" nil)
  ;;=> exit subprocess
#+end_src

** API Reference

*Namespace*: =jsonrpc=

All core functionality resides within the =jsonrpc= namespace. This includes the main connection class, message structures, and standard error specifications.

*** *Class* =jsonrpc::Conn=

The primary interface for managing JSON-RPC communication over standard I/O.

#+begin_src cpp
  Conn(Waker waker, 
       std::istream& input = std::cin, 
       std::ostream& output = std::cout, 
       std::ostream& error = std::cerr, 
       size_t max_pkg_size = kDefaultMaxContentLength);
#+end_src

- =waker=: A =std::function<void()>= called whenever a new message is pushed to the inbox. Use this to signal your main event loop to call =process_queue()=.
- =input/output/error=: The streams used for communication. On Windows, if these are =std::cin/cout/cerr=, they are automatically set to =_O_BINARY= mode.
- =max_pkg_size=: The maximum allowed =Content-Length=. Packets exceeding this are rejected to prevent memory exhaustion (Default: 16MB).

*Lifecycle Management*

- =void start()=: Launches the background reader thread. The thread continuously parses incoming LSP-framed packets and populates the inbox queue.
- =void stop()=: Signals the reader thread to exit and joins it. Note: In synchronous I/O scenarios, you may need to use =CancelIoEx= before calling this to unblock the thread.
- =bool is_running()=: Returns =true= if the reader thread is active.

*Method Registration*

- =void register_method(const std::string& name, RequestHandler handler)=: Registers a synchronous method.
  - =handler=: =std::function<json(const json& params)>=.
  - The return value is automatically sent back as a JSON-RPC result.
- =void register_async_method(const std::string& name, AsyncRequestHandler handler)=: Registers an asynchronous method.
  - =handler=: =std::function<void(Context ctx, const json& params)>=.
  - Execution completes only when =ctx.reply()= or =ctx.error()= is called.
- =void register_notification(const std::string& name, NotificationHandler handler)=: Registers a handler for notifications (requests without an =id=).
- =handler=: =std::function<void(const json& params)>=.

*Execution & Outbound*

- =void process_queue()=: Drains the inbox queue and executes the corresponding handlers. *Must be called on the thread where you want your business logic to run*.

- =void send_request(const std::string& method, const json& params, ResponseHandler callback)=: Sends a request to the client. The =callback= is executed when a response with the matching =id= is processed.

- =void send_notification(const std::string& method, const json& params)=: Sends a notification to the client.  

*** *Class* =jsonrpc::Context=

A handle passed to asynchronous handlers to provide a response at a later time.

- =void reply(json result)=: Sends a success response with the given result.
- =void error(int code, std::string message, json data = nullptr)=: Sends an error response.
- =std::optional<int> id()=: Returns the id of the current request.
- =bool is_notification()=: Returns true if the incoming message has no id.

*** *Exception* =jsonrpc::JsonRpcException=

Throw this inside a registered method to return a specific JSON-RPC error to the client.

- =Constructor=: =JsonRpcException(int code, const char* message, json data = nullptr)=.
- *Usage*: When caught by the Conn layer, it automatically generates a response using the provided code and message.

*** *Structs*: =Request=, =Response=, and =Error=

These are the data structures used internally for serialization and passed to handlers.

- =Request=: Contains =std::optional<int> id=, =std::string method=, and =json params=.

- =Response=: Contains =int id= and a variant of =json= (result) or =Error= (failure).

- =Error=: Contains =int code=, =std::string message=, and optional =json data=.
